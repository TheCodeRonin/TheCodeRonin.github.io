 PROJECT 01

PROJECT INFO




 THINGS TO CONSIDER

DATA: I definitely load a lot of data, more specifically on the INTERESTS page. However, I did reduce the image sizes and audio file sizes outside of code. Each image was edited in Affinity Photo. I used Foundation 6's interchange tool to display a smaller background image for the home page on mobile devices to avoid loading a higher resolution image. The home page background image is a PNG due to the transparency, so it has a larger amount of data to load that I couldn't avoid if I wanted to use this custom background. Every other image is a JPEG with reduced resolution. I have no idea what the best resolution to use is, but the original resolutions were too close to 2K. I reduced image sizes to things like 750px and 500px. Each audio sample was reduced from a WAV file to a 320 kbps MP3.

CSS: I did almost all of the styling in line, as I did not plan out how the website should look before I started adding a bunch of things. This made consistency confusing at first, but it did allow me to come up with specific styling on the spot. Next time I will use a lot more labels to streamline the styling process.

FORMATTING: I failed to comment enough to make my code understandable from an outside perspective. It became one of those things where it makes sense to me, but only because I created it. I will go back and comment/label all of my divisions to make customization easier. This would fix scalability problems in the future if I want to add or change the website.
